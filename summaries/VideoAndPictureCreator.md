## 一、项目概况

构建基于 Agnes AI API 的多模态内容生成工具集，包含图片生成（文生图/图生图）和视频生成（文生视频/图生视频）两大模块，通过命令行脚本封装 API 交互，输出结果统一保存到 output/ 目录。

## 三、优点总结

### 3.1 做得好的

1. **模块化组织**：image_gen/ 和 ideo_gen/ 各自独立，每个模块内含 SKILL.md、requirements.txt、scripts/，结构清晰，便于维护和扩展
2. **脚本封装良好**：ideo_gen.py 使用 argparse 提供完整命令行接口，支持 --dry-run 调试模式，参数校验充分
3. **异步流程设计合理**：创建任务→轮询状态→下载视频三段式，间隔10秒、最大300次重试，兼顾效率和稳定性
4. **num_frames 自动校正**：脚本内置 8n+1 规则自动对齐逻辑，降低用户使用门槛
5. **输出策略统一**：图片和视频统一存入 output/，版本化命名不覆盖已有资源
6. **SKILL.md 分层**：根目录统一 skill 总览 + 各模块详细文档，信息密度高且层次分明
7. **安全意识到位**：API Key 通过环境变量传递，严禁硬编码，智能体行为准则明确

### 3.2 技术亮点

- 视频下载支持流式写入（stream=True + iter_content），适合大文件
- 轮询接口使用 ideo_id + model_name 双参数，兼容上游原始 ID 场景
- 响应解析同时兼容 emixed_from_video_id 和 data.video_url 多种字段名

## 四、缺点与教训

### 4.1 核心教训

| # | 教训 | 根因 | 改进措施 |
|---|------|------|---------|
| 1 | **不要基于图片 API 的经验推断视频 API** | 两个模块的端点命名完全不同（/images/generations vs /videos），但初期盲目套用 | 遇到新 API 时优先查阅官方文档，而非凭直觉猜测 |
| 2 | **占位模型名必须替换为真实值** | gnes-video-1.0-flash 是占位符，上线前未验证 | 所有模型名必须在首次集成时通过 API 实际测试确认可用性 |
| 3 | **文档是第一手信息来源** | 官方文档已明确写出端点和参数，但初期选择盲测而非先读文档 | 任何 API 对接前，必须先阅读官方文档（如 wiki.agnes-ai.com），文档优先于试错 |
| 4 | **Python 异常类名大小写敏感** | equests.exceptions.httperror 应为 HTTPError，equestexception 应为 RequestException | 养成查阅官方文档的习惯，不要手写异常类名 |
| 5 | **异步 API 必须理解其语义** | 视频生成是异步任务，初始设计为同步等待，导致流程断裂 | 对接异步 API 时，第一时间确认是否存在轮询/回调机制 |
| 6 | **400 比 404 更有诊断价值** | GET /v1/videos/generation 返回 400（端点存在但方法不对），GET /v1/jobs/{id} 返回 404（端点不存在），应优先分析 400 的响应 | 建立错误码优先级：400/401/403 > 404 > 5xx，不同状态码对应不同排查方向 |

### 4.2 流程缺陷

1. **缺乏 API 健康检查步骤**：未在执行正式生成前先调用轻量接口（如模型列表、健康检查）验证连通性
2. **未建立端点映射表**：开发过程中端点反复试错，若早期整理一份对照表可大幅减少浪费
3. **SKILL.md 中的占位信息未及时清理**：ideo_gen/SKILL.md 初期写入了占位模型名和端点，后续修改时未全面同步

### 4.3 代码层面

1. **错误处理不够健壮**：poll_job_status 中捕获 RequestException 后直接重试，未区分网络错误和业务错误
2. **缺少进度反馈**：轮询期间仅打印状态和进度百分比，未显示预估剩余时间
3. **硬编码超时值**：DEFAULT_TIMEOUT = 600 对所有场景一刀切，短提示词和复杂场景应有不同策略
4. **未处理并发冲突**：多次生成共享 ideo_1.mp4 文件名，无版本号递增机制

## 五、最佳实践清单（未来项目复用）

### 对接新 API 前

- [ ] 阅读官方文档（wiki/llms.txt），记录所有端点和参数
- [ ] 建立端点-方法-参数对照表
- [ ] 验证模型名是否真实可用（非占位符）
- [ ] 确认同步/异步语义，设计对应流程
- [ ] 设置最小可行测试用例（一个 prompt，最短时长）

### 编码规范

- [ ] 异常类名严格对照官方文档拼写（首字母大写）
- [ ] 所有魔法字符串（URL、模型名、超时值）提取为常量
- [ ] argparse 参数与 API 参数一一对应，避免遗漏
- [ ] 输出文件名加入时间戳或版本号，防止覆盖

### 安全规范

- [ ] API Key 仅通过环境变量传递，不写入代码/文档/日志
- [ ] 智能体行为准则中明确"主动索要"机制
- [ ] 用户提供的凭据仅存于当前会话内存

### 错误排查优先级

1. 401/403 → 检查认证和权限
2. 400 → 检查参数格式和必填项
3. 404 → 端点路径或方法错误
4. 503 → 模型不可用或服务繁忙
5. 超时 → 异步任务未正确处理轮询
